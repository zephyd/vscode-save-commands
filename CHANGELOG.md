# Change Log

All notable changes to the "save-commands-enhanced" extension will be documented in this file.

## [1.2.1]
- **Drag and Drop**: Resolved regression where empty folders couldn't be targeted; re-established robust ID-based transfers.
- **Workflow**: Implemented double-click to edit to prevent interference with single-click drag operations.
- **Multi-line Commands**: Upgraded command input to a textarea with `Ctrl+Enter` for quick submission.
- **UI/UX**: Redesigned the add/edit form with a premium Gemini-style terminal logo and refined icon interactive feedback.
- **Dynamic Helper**: Reimagined the dynamic parameter assistant with a clean `{ }` icon and optimized footer layout.

## [1.2.0]
- **Run Folder**: Added ability to run all commands in a folder with a configurable joiner.
- **Drag and Drop**: Added support for reordering and moving items between folders.
- **Unit Testing**: Established a comprehensive unit test suite with VS Code API mocking.
- **Performance**: Improved placeholder resolution logic.
